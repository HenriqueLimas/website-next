# Why Marko

## Performance

- Streaming
- High throughput
- Targeted compilation
- Resumability
  - Avoid redoing server work
  - Avoid bundling server only operations
- Fine grained reactivity

## DX

- HTML but better
- Low boilerplate
- Highly declarative
- Terse yet readable
- Composable and extensible

## The Marko Mindset

- Use the platform when possible
- Localization of Concerns
